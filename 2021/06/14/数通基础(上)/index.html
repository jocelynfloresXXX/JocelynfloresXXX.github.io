# 01【网络基础知识】

1. 概念：

如下如下

  • 交换：Switching
  • 单播：Unicast
  • 组播：Multicast
  • 广播：Broadcast
  • 广播域：Broadcast Domain

2，交换机：

如下如下

  • 交换机的每一个接口是一个冲突域。
  • 交换机的整机是一个广播域（也就是说连接到交换机上的所有节点是一个广播域 [LAN]）。
  • 广播是一个负担（所以要减小广播域的大小）。
  • 交换机的基本功能：
  • 数据帧的交换。
  • 终端用户设备的接入。
  • 基本的接入安全功能。
  • 广播域的隔离（VLAN）。
  • 二层链路的冗余，防环及负载均衡。

3，路由器：

如下如下

  • 路由器的基本功能：
  • 隔离广播，实现跨三层的数据互访。
  • 路由协议的支持，维护路由表。
  • 路径选择及数据转发。
  • 广域网接入，地址转换及特定的安全功能。
  • 流量管理

4，OSI 参考模型 (写了好多遍了 ——_——！！！)：

如下如下

  • OSI 模型的目的：
  • 规范不同系统的互联标准，使两个不同的系统能够较容易的通信，而不需要改变底层的硬件或软件的逻辑。
  • OSI 模型分为七层：
  • 由上到下分别为物理层，数据链路层，网络层，传输层，会话层，表示出，应用层。
  • 高层（7-5）：负责主机之间的数据传输。
  • 底层（4-1）：负责网络数据传输。
  • ①：应用层：
  • 各种应用程序，协议。
  • 详解：为应用软件提供接口，使应用程序能够使用网络服务。
  • 常见的应用层协议：
  • http (80)，ftp (20/21)，smtp (25)，pop3 (110)，telnet (23)，dns (53) 等。
  • ②：表示层：
  • 数据和信息的语法转换内码，数据压缩解压，加密解密。
  • 详解：
  • 常见的标准如：
  • ASCII
  • JPEG
  • ....
  • ③：会话层：
  • 为通信双方定制通信方式，并创建，注销会话。
  • 详解：
  • 在设备或节点之间提供会话控制。
  • 它在系统之间协调通信过程。
  • ④：传输层：
  • 提高可靠或者不可靠的端到端传输。
  • 详解：
  • 负责将来自上层应用程序的数据进行分段或者重组，并将他们组合为同样的数据流形式。
  • 提供端到端的数据传输服务。
  • 传输层协议：TCP&UDP。
  • ⑤：网络层：
  • 逻辑寻址，路由选择。
  • 详解：
  • 定义了逻辑地址 (三层地址)
  • 分组寻址，负责将分组数据从源端传输到目的端。
  • 路由选择，维护路由表。
  • ⑥：数据链路层：
  • 将分组数据封装成帧，提供节点到节点方式的传输，差错校验。
  • 详解：
  • 在不可靠的物理链路上，提供可靠的数据传输服务，把帧从一跳（节点）移动到另一跳（节点）。
  • 局域网的数据链路层协议有以太网，令牌环网等。
  • 广域网数据链路层协议有：PPP，HDLC，Frame Relay 等。
  • ⑦：物理层：
  • 在媒介上传输比特流，提供机器的和电气的规约。

# 02【TCP/IP 参考模型】

1，TCP&UDP：

如下如下

  • TCP (传输控制协议)：
  • 属于面向连接的网络协议。
  • 可靠传输。
  • 流控及窗口机制。
  • UDP (用户报文协议)：
  • 属于无连接的网络协议。
  • 不可靠传输。
  • 尽力而为的传输。

2，TCP&UDP 端口号：

如下如下

  • 源端口随机分配，目标端口使用知名端口。
  • 应用客户端使用的源端口号一般为系统中未使用的且大于 1023。
  • 目标端口号为服务器端应用服务的进程，如 telnet 为 23.

3，窗口大小：

如下如下

  • 决定了在收到确认前可以发送的字节数 (win=?)。

4，ARP 协议：

如下如下

  • 通告广播请求（增加网络负载）。
  • 不可靠，不安全。
  • 将 IPV4 地址解析为 MAC 地址。
  • 维护 ARP 映射的缓存。
  • 查看 ARP 表：arp -a
  • 查看路由路径：tracert

# 03【IP 子网划分】

1，VLAM：

如下如下

  • 为什么要划分子网？
  • 一个 B 类地址用于一个广播域，地址浪费。
  • 广播域太庞大，一旦发生广播，内网不堪重负。
  • 将一个网络号划分成多个子网，每个子网分配一个独立的广播域。
  • 广播域缩小，网络规划更加合理。
  • IP 地址得到了合理利用。

2，子网：

如下如下

  • 新加子网数为：2 的 M 次方（M 为借的主机数量）

3，IP 地址类别：

如下如下

  • 私有地址：
  • 10.0.0.0——10.255.255.255
  • 172.16.0.0——172.31.255.255
  • 192.168.0.0——192.168.255.255
  • 如果私有 IP 地址用户想要访问公网，则需要使用 NAT 网络地址转换技术。
