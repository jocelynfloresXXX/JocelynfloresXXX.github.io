1，AD 单臂部署介绍：

如下如下

  • 1）配置 AD 部署模式时不需要选择什么部署模式，AD 以什么方式接入就是什么部署模式。
  • 2）AD 单臂部署不需要改变原有网络结构，可实现服务器负载，入站链路负载功能，不支持出站链路负载。
  • 3）单臂部署时，必须连接 WAN 口到交换机（WAN 口可与服务器同网段，不同网段时需要保证 AD 与服务器路由可达）。
  • 4）服务器发布资源到公网，需要在边界出口设备做端口映射给 AD 设备（映射分为全映射或相关端口映射）。
  • 5）拓扑：
  • 6）案例配置：
  • ①：通过设备管理口（10.252.252.252/10.254.254.254）登录设备，配置网口。
  • ②：配置 AD Wan 口信息，路径：【网络配置】-【网络接口】配置网口 (WAN) 地址。
  • ③：若通过 WAN 口 IP 登录设备，需要开启远程维护功能（7.x 版本以下要同时启用 WAN 口入站路由转发）。
  • ④：AD 启用源地址转换（服务器看到的源 IP 是 AD 的，所以回包也回给 AD）
  • ⑤：若服务要发布到公网，要在边界设备上做端口映射给 AD 设备。
  • ⑥：总结：
  • WAN 口设置。
  • 远程维护和 WAN 口入站路由转发。
  • 源地址转换。
  • 端口映射。
  • 7）单臂部署 ADWAN 口必须选择 WAN 属性。

2，AD 网关（路由）部署介绍：

如下如下

  • 1）一般把 AD 设备放在网关出口的位置，做路由器来使用，设备直接接入网络边界，启用 NAT 代理上网等功能，防火墙做透明模式。
  • 2）可实现的功能有：出入站链路负载，服务器负载。
  • 3）拓扑：
  • 4）案例配置：
  • ①：客户有联通，电信两条出口线路，希望实现代理内网上网和智能选路，联通走联通，电信走电信。（防火墙透明部署）
  • ②：配置 AD 外网口（WAN 口）和内网口（LAN 口）地址。
  • ③：内网有多网段，要添加静态路由回指给设备下接的核心交换机。
  • ④：配置代理上网（SNAT）
  • ⑤：配置智能路由。
  • ⑥：总结：
  • 配置 LAN 口地址。
  • 配置 WAN 口地址和网关。
  • 配置静态路由。
  • 配置代理上网。
  • 配置智能路由。

  • 6）如果内网是二层环境，则不需要配置静态路由，将内网设备的网关指向 AD 的 LAN 口即可。

3，AD 设备网口设置：

如下如下

  • 1) 端口聚合：
  • 将多个物理网口当作一个单一的逻辑接口来处理，提高更高的带宽，吞吐量和可恢复性技术主要特点有：
  • 增加带宽：带宽相当于接口的带宽总和。
  • 增加冗余：组内不是所有接口都故障，两个设备之间仍可通信。
  • 负载均衡：流量可以在接口上按照算法自动进行负载均衡。
  • 2）多个物理口聚合一个逻辑口的策略有：
  • 哈希，轮询，802.3ad，冗余双网卡。
  • 3）vlan 子接口场景：
  • AD 旁路部署时，设备只接一根线，但这个线路上配置多个网段的 IP（IP 属于不同的 VLAN 中），这时可根据 IP 所属 VALN 信息来配置对应的 VALN 子接口和交换机做 TRUNK 对接。（VLAN ID 要与对端对接的设备 VALN 号相同）
  • 4）端口桥接：
  • 复杂的网络中，要求 AD 能将两个口划分到一个广播域（AD 设备的多个网口组合成一个二层交换机）。
  • 5）内网客户部署两台防火墙，主备，AD 部署网络出口，NET2 与 NET3 接口 IP 一样（同一 VLAN），防火墙发送的数据通过 NET2 和 NET3 均能与该 VLAN IP 通信。

4，AD 集群模式部署：

如下

  • 1) 介绍：
  • 一个服务只运行在一台主机，当主机故障时切换到备机继续提供服务，多个服务可以分布运行在不同的设备之上互作为备份。
  • 2）AD HA 口作用：
  • 设备间同步配置，可定时进行心跳检测，但设备上作为 HA 口的网口顺序一致。
  • 3）浮动 IP：
  • 虚拟服务 / 其他业务需要访问的 IP，可以根据情况漂浮在不同设备间，一个浮动 IP 同时只在一台设备上生效。
  • 4）静态 IP：
  • 对服务器 / 链路做健康检测时用到的 IP，每台设备业务口必须配置一个静态 IP。
  • 5）集群准备：
  • ①：IP 规划：
  • HA 口 IP，两台设备 HA 口都需要配置 IP（同网段可通信，网段可随意与内网不冲突即可）
  • 浮动 IP & 静态 IP：相当于单台设备时业务口配置的 IP，能与服务器 & 外网通信。区别在于浮动 IP 可能会在不同的设备生效，静态 IP 只在一台设备生效，浮动 IP 用于虚拟服务接入，静态 IP 用于健康检查。
  • 5）主备心跳口：
  • 集群间的状态维护，会话同步等都依赖主备心跳口完成。
  • 主备心跳口端口不能冲突，业务口也可作为心跳口（心跳口为管理口时无法进行会话同步）。

5，AD 全局负载（单节点）：

如下

  • 1) LDNS：线路开通时，运行商告诉客户的 DNS 服务器。
  • A 记录：指定主机名（域名）对应的 IP 地址。
  • NS 记录：域名服务器记录，该域名由哪个 DNS 服务器来解析。
  • AD 当作 DNS 服务器，收到外网域名请求时，通过算法计算出最优链路将 IP 返回。
  • 2）AD 设备上 DNS 映射是将域名和对应的虚拟 IP 池对应起来。
  • 虚拟 IP 池：是一个域名解析的对应 IP 地址的逻辑分组（一个或多个 IP）有自己的调度策略，DNS 映射将域名与虚拟 IP 池对应起来，调度到虚拟 IP 池后，根据虚拟 IP 池的策略（有：静态就近性，轮询，加权轮询等）返回对应 IP 地址。
  • 3）调度策略推荐使用的有：
  • 若多条同一运行商的链路，考虑使用轮询 / 加权轮询 / 哈希。
  • 不同运行商的链路，考虑使用静态就近性。
  • 入站流量不大且源 IP 范围大多是国外，考虑使用动态就近性。
  • 链路之间带宽差异大，考虑使用加权最小连接 / 加权轮询。
  • 4）AD DNS 记录：
  • DNS 映射只针对 A 记录 / AAAA（IPv6）记录的，AD 作为 DNS 服务器需要其他记录时可在 AD 上添加相关 DNS 记录。
  • 同一域名配置了 DNS 映射和 A 记录，DNS 映射优先于 A 记录。

5，应用负载高级配置：

如下

  • 1) 前置调度策略：
  • 可根据源 IP，目的 IP，TCP 流，HTTP 请求头 / 行等进行调度，改写调度，关闭连接，返回指定内容等，使用调度与服务相关（有：源 IP，HTTP 请求头 / 行，证书变量进行调度，改写，返回指定内容等。）
  • 2）原理：
  • 符合调度条件的请求始终调度到某一台或多台服务器上（前置调度策略优先于虚拟服务关联的节点池调度策略）。
  • 3）基于源 IP 进行分发：客户想让不同的客户端访问时调度到不同的服务器，应用场景广泛。
  • 4）基于 URL 进行分发：不同的 URL 调度到不同的节点（如：客户有一个域名不同的路径需要调度到不同的服务器）。
  • 5）HTTP 头部改写（请求改写 & 应答改写）：
  • HTTP 头部有
  • ：请求行，应答行，请求头部，应答头部
  • 改写原理：
  • 数据经过 AD 后，根据业务需求对请求 / 应答数据进行相应的改写再发送给客户端，达到正常业务访问的需求。
  • 改写策略在前置调度中引用，先匹配上前置调度策略才能匹配改写策略。
  • 6）HTTP 改写 - 调度（HTTP 方式才有）：
  • 调度分为：首个请求（只有首个请求会做调度），每个请求（默认选择，每个 HTTP 对话都会调度）。
  • 请求改写：在 AD 收到客户端的请求后，发给服务器之前，AD 改写请求头部对客户端是透明的。
  • 应答改写：AD 收到服务器的回应之后，发送给客户端之前，客户端一定能看出效果。
  • 7）HTTP 改成 HTTPS：
  • 应答 302 跳转，Location 中 http 改写成 https。
  • 应答实体中 http 改写 https。
  • URL 中 80 等都需要去掉。

6，智能路由基本原理

如下

  • 1) 原理：
  • 出口有多条外网线路，根据源 IP，端口，链路选择策略等条件设定路由，来确定数据从哪个 WAN 口转发实现智能；路由选择功能，目的是对多线路实现流量分隔，优化带宽等目的（可单独使用，推荐和 DNS 代理一起使用）。
  • 2）链路选择的策略有：
  • 轮询：轮流调度已选择的链路，均等。
  • 带宽比例：根据每条链路的带宽比列，选择链路的机会与链路上下行带宽之后成正比。
  • 加权最小流量：选择带宽利用率最小的链路。
  • 动态就近性：根据探测目的端得到的 RTT（往返时延）选择 RTT 最小的线路。
  • 加权源 IP 哈希：按照每条链路的带宽比列，不同源 IP 选择链路的机会与链路上下行带宽之和成正比。
  • 3）智能路由的会话保持：
  • 默认会按照源目地址进行会话保持（可以改会话保持的子网掩码和超时时间）。通过子网掩码和目的 IP 与运算得到一个网络号来形成一条出站会话保持记录。
